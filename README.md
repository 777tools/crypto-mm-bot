# 🤖 crypto-mm-bot — bitbank マーケットメイキングBot

GMO と bitbank の価格差（乖離）を利用して、ETH/JPY で自動マーケットメイキング（指値の買い・売りを並べて利ざやを狙う）を行う自動売買ボットです。4段階の相場判定・シミュレーションモード・Web管理画面つき。

---

## ⚠️ 最初に必ずお読みください

- 本ツールは**教育・研究目的**です。投資助言ではありません。実取引は**自己責任**で行ってください。
- 暗号資産の自動売買は**損失のリスク**があります。まず必ず**シミュレーションモード**で動作を確認してください。
- APIキーは**あなた自身の bitbank アカウント**のものを使います。第三者に渡さないでください。

### 🚨 セキュリティ重要通知（開発履歴について）

本プロジェクトの開発中、**APIキーがソースコードに直書きされた旧版ファイルが、開発者のローカルgit履歴（未公開のオブジェクト）に一時残存していました**。公開リポジトリ（GitHub）には一度もpushされていませんが、**過去にこのBotの初期版を使ったことがある方・該当のAPIキーの持ち主は、bitbankの管理画面で直ちにキーを失効（削除）し、新しいキーを再発行してください**。キーの失効・再発行はbitbankへのログインが必要なため、ツール側では行えません。

## 🖥️ どこに設置するか

このボットは **24時間動かし続ける**必要があります（10秒ごとに価格を見て売買するため）。
そのため、**ご自身が契約したレンタルサーバー（VPS）に設置してください。**

- 例：さくらのVPS / ConoHa VPS / Xserver VPS など、Python が動くレンタルサーバー
- お試しだけなら、自宅PCでも動きます（PCの電源を切ると停止します）

> 販売元のサーバーは使いません。必ずご自身のレンタルサーバー・PCで動かしてください。

## 🚀 かんたん設置ガイド（4ステップ）

### 1. ファイルを置く
契約したレンタルサーバーに、このフォルダ一式をアップロードします。

### 2. 管理画面の初期設定をする（初回のみ）
ブラウザで `https://あなたのサーバー/web/` を開くと、**初回だけ管理用IDとパスワード（8文字以上）の設定画面**になります。
※ 固定の初期パスワードはありません。必ずあなた自身のパスワードを設定してください。

### 3. APIキーを設定する
管理画面の「🔑 APIキー設定」から bitbank の APIキーとシークレットを登録します（サーバーの `.env` に保存され、画面には表示されません）。

手動で設定する場合は `.env.example` をコピーして `.env` を作ります：

```bash
cp .env.example .env
chmod 600 .env
```

```
BITBANK_API_KEY=あなたのAPIキー
BITBANK_API_SECRET=あなたのAPIシークレット
```

> bitbank の APIキーは、bitbank にログイン →「API」メニュー → 新規発行 で取得できます。
> **「資産の参照」と「注文（新規・取消）」権限にチェック**を入れて発行してください。

### 4. 必要なものを入れて動かす
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**まずはシミュレーション（お金を使わずテスト）：**
```bash
python3 bot_v2.py --mode sim    # または python3 demo.py（同じもの）
```

**本番（実際に売買）：**
```bash
python3 bot_v2.py --mode live
```

本番モードは以下を**全て満たさないと起動しません**（フェイルクローズ）：
1. `.env` に APIキー/シークレットが設定済み
2. 管理画面で「本番モードを有効化」を明示的にON（`live_confirmed`）
3. 起動のたびに確認入力
4. 安全上限（最大保有・日次損失・注文額など）が正常範囲

## ⚙️ 設定（bot_config.json / 管理画面「売買ルール」）

設定は `bot_config.json` に保存され、Botが起動時に読み込みます。**範囲外の値は起動を拒否します**（管理画面側でも同じ範囲チェックあり）。

| 設定 | 意味 | 初期値 | 許容範囲 |
|---|---|---|---|
| `symbol` | 通貨ペア | eth_jpy | 主要ペアのみ |
| `order_size` | 1回の注文量 | 0.05 ETH | 0.001〜1 |
| `interval` | 価格チェック間隔（秒） | 10 | 3〜300 |
| `spread_range` | レンジ相場のスプレッド | 300円 | 10〜100000 |
| `spread_trend` | トレンド相場のスプレッド | 800円 | 10〜100000 |
| `spread_threshold` | 乖離しきい値 | 300円 | 0〜100000 |
| `crash_threshold` | 急落判定（%） | 3% | 0.1〜50 |
| `stop_loss_rate` | 損切り幅 | 0.02（2%） | 0.001〜0.5 |
| `max_position` | 最大保有 | 0.2 ETH | 0.001〜10 |
| `daily_loss_limit` | 日次損失の上限 | 10000円 | 100〜1000万 |
| `max_order_jpy` | 1注文の上限額 | 100000円 | 100〜1000万 |
| `max_open_orders` | 同時注文数の上限 | 4 | 1〜20 |
| `post_only` | メーカー注文のみ | true | - |

## 🛡️ 本番モードの安全装置

- **残高確認**：起動時と毎ループで資産を確認。取得失敗時は新規発注せず自Bot注文を取消（フェイルクローズ）
- **自Bot注文のみ管理**：発注した注文IDを台帳（`own_orders.json`）に記録し、**自分の注文だけ**を取消します。手動注文・他Botの注文には一切触れません
- **取消失敗時は新規発注禁止**：取消に失敗した注文がある間は新たな注文を出しません
- **日次損失の上限**：1日の含み損益が上限に達したら全停止
- **上限群**：最大保有・1注文上限額・同時注文数・post_only
- **安全な停止**：SIGTERM/SIGINT を受けると自Botの未約定注文を取消してから終了。取消に失敗した場合は注文IDを明示して警告します

## 🧪 4段階の相場判定

1. **乖離チェック** — GMOとbitbankの価格差が十分あるか
2. **ボラティリティ** — 値動きが激しすぎないか
3. **レンジ/トレンド** — 相場の種類を判定してスプレッドを調整
4. **急落ガード** — 暴落時は自動で発注を止める
